# 全部成品工單物料需求 API

- **路徑**: `/api/orders/finished/requirements`
- **方法**: `GET`
- **說明**: 一次性獲取系統中所有 1 開頭 (成品) 工單的物料需求明細。此資料直接從後端快取 (Cache) 中讀取，因此查詢效能極高，無需對資料庫或 Excel 進行多次 I/O 操作。

## 回傳格式 (JSON)

回傳結果為 JSON 格式，包含查詢狀態、回傳的工單總數量，以及詳細的工單物料資料字典。

```json
{
  "status": "success",
  "count": 125,
  "data": {
    "10000001": [
      {
        "物料": "A00123",
        "物料說明": "測試物料 A",
        "需求數量 (EINHEIT)": 5.0,
        "領料數量 (EINHEIT)": 2.0,
        "未結數量 (EINHEIT)": 3.0,
        "需求日期": "2024-03-15",
        "unrestricted_stock": 10.0,
        "inspection_stock": 0.0,
        "order_shortage": 0.0
      },
      ...
    ],
    "10000002": [
      ...
    ]
  }
}
```

## 欄位說明

### 根節點 (Root)
*   **`status`** (String): API 處理狀態，成功時為 `"success"`。
*   **`count`** (Integer): 本次成功回傳的 1 開頭 (成品) 工單總數。
*   **`data`** (Object): 以「工單號碼」(字串格式) 作為鍵 (Key)，對應的「物料需求明細陣列」(Array) 作為值 (Value) 的字典集。

### 物料需求明細陣列 (`data` 內的物件)
單一工單的物料需求，陣列內的每一個物件包含以下欄位：
*   **`物料`** (String): 物料料號。
*   **`物料說明`** (String): 物料的中文名稱或規格說明。
*   **`需求數量 (EINHEIT)`** (Float): 此工單對該物料的總計劃需求數量。
*   **`領料數量 (EINHEIT)`** (Float): 已實際領用的數量。
*   **`未結數量 (EINHEIT)`** (Float): 尚未領取的數量 (需求數量 - 領料數量)。
*   **`需求日期`** (String): 預計需使用該物料的日期，格式為 `YYYY-MM-DD` (若無日期則回傳空字串 `""`)。
*   **`unrestricted_stock`** (Float): 該物料在系統中目前的「未限制庫存數量」。
*   **`inspection_stock`** (Float): 該物料在系統中目前的「品質檢驗中庫存數量」。
*   **`order_shortage`** (Float): 若大於 0 表示目前的可用庫存無法滿足此單筆明細的未結數量。計算方式為：`(未結數量 - (未限制庫存 + 品檢中庫存))`。

## 錯誤回應

若伺服器發生異常或後端資料 (Cache) 尚未載入完成時，會回傳錯誤訊息。

*   **HTTP 500 Internal Server Error**
```json
{
  "error": "資料尚未載入"
}
```
或
```json
{
  "error": "一個內部伺服器錯誤發生了"
}
```

## 使用場景
- **批量查詢與報表匯出**：前端若需要一次性下載或顯示所有成品工單的備料現況，調用此 API 將能大幅降低單獨查詢每張工單的網路請求次數及時間。
- **整體缺料預警儀表板**：可藉由前端一次性獲取資料後，統計所有 1 開頭工單的整體 `order_shortage` 狀況。
